Реферат
Пояснительная записка курсового проекта содержит 58 страниц пояснительной записки, 44 иллюстрации, 7 источников литературы, 5 приложений.

ASP.NET, MICROSOFT IDENTITY, GIT, CLOUD MONGODB, TYPESCRIPT, REACT, DOCKER, DOCKER HUB, AZURE WEB APP, .NET 5.

Целью курсового проекта является создание одностраничного веб-приложения для обеспечения работы и управления мото-сервиса, а также которое может использоваться как шаблон для других приложений с классической иерархией управления Администратор-Работник-Клиент. Неавторизованный пользователь может зарегистрироваться, авторизоваться, просматривать информацию о сервисе, видеть отзывы других клиентов, а также восстановить утерянный пароль. Авторизованный пользователь может иметь роль Клиент, Работник или Администратор. Любой клиент может оставлять заявки для предоставления сервиса. Администратор управляет ролями, может следить за исполнением заявок, а также назначает заявки работникам. Работники могут изменять статус заявок, назначенных ему. После закрытия заявки у клиента появляется возможность оставить отзыв, привязанный к работнику.  
В первой главе проводится аналитический обзор прототипов по тематике курсового проекта и содержится описание технологий, использованных во время выполнения проекта.
Вторая глава посвящена процессу разработки архитектуры проекта.
В третьей главе описывается процесс разработки проекта, реализации поставленных задач.
Четвёртая глава посвящена тестированию приложения.
